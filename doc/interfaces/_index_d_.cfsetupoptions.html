Optional langSrcMap

langSrcMap: object
  • Defined in index.d.ts:1015

如果觉得核心包中逻辑相关的提示文字多语言翻译不满足你的要求，或者支持的语言不够多 你可以配置该属性指定使用自己准备的翻译文本，这个配置的每个属性的属性名是语言代号， 其属性值是该语言翻译文本（json格式）所在的网络地址 比如，核心模块当前仅提供了中文（zh_cn）和英文（en_us）的多语言文本 现在你对核心模块的中文翻译是满意的但是对核心模块的英文的翻译内容不满意， 这样你就可以配置该属性如下内容

(我们假设你将多语言文本放到了下面这个网址中)

{
 "en_us": 'https://yousite.com/your-os-client/es_us.json'
}

以上配置会覆盖掉默认的英文翻译，当进行英文翻译时，会只用你提供的这本json作为翻译依据， 核心模块提供的英文翻译会被丢弃

如果在此基础上,同时你希望有日文版的多语言翻译，这时你可以添加配置日文翻译文本

{
 "en_us": 'https://yousite.com/your-os-client/es_us.json',
 "ja_jp": 'https://yousite.com/your-os-client/ja_jp.json',
}

以上的配置覆盖了英文翻译，同时新增了日文翻译。

每本json要准备的数据格式在LANG_ZH_CN和LANG_EN_US中做出了示范.

各种语言的代号可参照百科全书

我们统一使用xx_xx小写字母加下划线形式的代号

Optional setter

setter: CFPropSetFn
  • Defined in index.d.ts:916

属性设置辅助方法 由于UI中绑定了核心提供的数据结构的某些值，UI中进行操作后回调了核心提供的回调函数 核心中的逻辑会更改数据结构中的这些值，核心逻辑中默认是直接更改这些值的。 这样的话，大部分框架的UI模板引擎是无法觉察到这个更改的，从而UI上的内容不会正常更新。 这时候可以通过设置该方法来使得模板引擎觉察到内容的更改达到更新页面的目的。

举例：UI中渲染选项时背景颜色绑定了CFOption.selected属性， 选中的选项（option.selected为true）渲染成红色， 未选中的选项（option.selected为false）渲染成灰色， 现在某个选项初始时是未选中的，根据绑定规则，渲染成了灰色，现在点击该选项，会回调核心 逻辑中提供的选项点击回调函数，在该函数中会将option.selected改为true， 但是UI模板并不知道这个属性发生了变化，所以依然渲染成了灰色，这时就需要用的这个配置方法。 如使用emberjs的时候我们将该方法配置为EmberObject的setProperties方法，

  import {setProperties} from '@ember/object';
  options.setter = setProperties

使用angularjs(version 1.x)是我们可以使用$scope.digest或$timeout来触发变化

 options.setter = (toObj, propsObj) => {
    Object.assign(toObj, propsObj);
    $timeout(() => {})
    // 或者
    $rootScope.digest();
 }

使用angular(version 2+)时，我们可以使用app.tick()来触发变化

 options.setter = (toObj, propsObj) => {
    Object.assign(toObj, propsObj);
    app.tick()
 }

使用react时，我们可以使用顶层的setState来触发变化

 options.setter = (toObj, propsObj) => {
    Object.assign(toObj, propsObj);
    this.setState({});
 }

每次更改都会调用这个方法,可能每次操作会触发多次调用,可能会有性能影响 所以可以考虑使用debounce过的方法,来作为回调方法,减少触发更新的次数,减少性能上的影响. 也可以不设置这个方法,而是使用CFClient.EventHub监听SET_PROPS事件, 然后在个事件的回调中,驱动框架更新状态

import {EventHub} from "@choiceform/os-client-core";
EventHub.on('SET_PROPS', () => {
  // 这里放置驱动框架更新的代码
  // 如:this.setState();
  // 如:app.tick();
  // 如:$rootScope.digest();
  // 等等...
})

具体的使用方式在各个框架的client-demo中也会展示。 如果你使用的模板引擎能自动检查到外部数据的变化，则可以不配置该方法

    严重错误消息提示方法 如尝试下载某份问卷进行作答，但这份问卷已经下线，则不会得到问卷内容 而是会发出一个类似”问卷已下线“的错误消息 通过配置该方法来决定在UI中展示这些错误消息。 发生任何错误后，不允许继续操作。 如果没有配置该方法，消息会在调试控制台中以错误消息方式输出。
